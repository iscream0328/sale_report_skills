# Brand Image Matching

This is a workspace-only skill for `<project-root>`. Use it only when `skill_ver/scripts/build_brand_url_matches.py` and `skill_ver/data/portfolio_all_index.json` exist.

## Quick Start

Run from the project root:

```bash
python3 skill_ver/scripts/build_brand_url_matches.py \
  --url "https://example-brand.com/" \
  --instagram-folder "script/ig_downloads/downloads/example_profile" \
  --brand-slug example_brand \
  --max-pages 12 \
  --max-downloads 100
```

If Instagram is needed, download it first with the local guide:

```bash
./script/ig_downloads/download_ig.command profile_name 20 chrome
```

Then pass the resulting folder:

```bash
python3 skill_ver/scripts/build_brand_url_matches.py \
  --url "https://example-brand.com/" \
  --instagram-folder "script/ig_downloads/downloads/profile_name" \
  --brand-slug example_brand
```

## Inputs

- `--url`: Brand store, product list, collection, campaign, lookbook, editorial, or project URL. Repeat for multiple URLs.
- `--instagram-folder`: A gallery-dl download folder created by `script/ig_downloads/download_ig.command`.
- `--source-html`: Saved HTML for pages that cannot be fetched directly.
- `--brand-slug`: Stable output slug. Use lowercase ASCII with underscores when possible.
- `--run-id`: Optional stable run id when regenerating a known viewer.
- `--instagram-max-images-per-post`: Defaults to 2 so one carousel does not dominate the brand mood.

## Outputs

The script writes to:

```text
skill_ver/data/brand_runs/{brand_slug}_{run_id}/
```

Core files:

- `brand_source_images.json`: All analyzed brand source image records.
- `brand_source_summary.json`: Product/campaign selections and observed mood distribution.
- `portfolio_recommendations.json`: Similar 10 and whitespace 10 recommendations with evidence.
- `index.html`: Human review surface with source links, recommendation strategy, tag guide, and comparison cards.
- `source_images/`, `source_thumbnails/`: Local image assets for the review page.

Optional outreach files, generated by `skill_ver/scripts/build_brand_outreach_assets.py`:

- `outreach_assets/outreach_assets.json`: Structured cold email and proposal data.
- `outreach_assets/cold_email_draft.md`: Editable Korean cold email draft.
- `outreach_assets/mini_proposal.html`: Internal proposal builder where the 담당자 selects from the 10 similar and 10 whitespace portfolio candidates, clicks `제안서 생성`, downloads SECTION-only HTML, or downloads the pre-generated landscape PDF.
- `outreach_assets/proposal_sections_landscape.pdf`: A4 landscape PDF generated without opening the print UI by `skill_ver/scripts/export_proposal_sections_pdf.py`.

## Recommended Workflow

1. Confirm you are in `<project-root>`.
2. If Instagram should be included, follow `script/ig_downloads/IGDOWNGUIDE.md` and download with a logged-in browser cookie.
3. Run `build_brand_url_matches.py` with the store URL and/or Instagram folder.
4. Validate:
   ```bash
   python3 -m py_compile skill_ver/scripts/build_brand_url_matches.py
   python3 -m json.tool skill_ver/data/brand_runs/<run>/brand_source_images.json >/dev/null
   python3 -m json.tool skill_ver/data/brand_runs/<run>/portfolio_recommendations.json >/dev/null
   ```
5. Browser-test `index.html` with Playwright when layout or interactions changed. Check desktop and mobile widths for image loading, no horizontal overflow, clickable source images, and tooltip behavior.
6. Report the viewer path, candidate count, selected product/campaign counts, similar/whitespace counts, and any warnings.

## Outreach Assets

After the matching run is complete, use the `proposal-builder` skill for the cold email, mini proposal, SECTION HTML, and landscape PDF export workflow. The core commands are:

```bash
python3 skill_ver/scripts/build_brand_outreach_assets.py \
  --run-dir skill_ver/data/brand_runs/<run> \
  --brand-name "브랜드명" \
  --contact-name "브랜드 담당자님"
```

Default selection uses the top 3 similar recommendations and top 3 whitespace recommendations.
When the user has chosen specific images, pass portfolio IDs:

```bash
python3 skill_ver/scripts/build_brand_outreach_assets.py \
  --run-dir skill_ver/data/brand_runs/<run> \
  --brand-name "브랜드명" \
  --similar-ids P081 P048 P055 \
  --whitespace-ids P030 P083 P117
```

Validate:

```bash
python3 -m py_compile skill_ver/scripts/build_brand_outreach_assets.py
python3 -m py_compile skill_ver/scripts/export_proposal_sections_pdf.py
python3 -m json.tool skill_ver/data/brand_runs/<run>/outreach_assets/outreach_assets.json >/dev/null
```

To create a direct-download landscape PDF from the selected defaults:

```bash
python3 skill_ver/scripts/export_proposal_sections_pdf.py \
  --run-dir skill_ver/data/brand_runs/<run>
```

To export a specific selection:

```bash
python3 skill_ver/scripts/export_proposal_sections_pdf.py \
  --run-dir skill_ver/data/brand_runs/<run> \
  --similar-ids P081 P082 P048 \
  --whitespace-ids P030 P083 P084
```

Browser-test `outreach_assets/mini_proposal.html` with Playwright when layout changes. Verify that it shows 10 similar candidates and 10 whitespace candidates, `제안서 생성` renders selected portfolio pages with at most 3 images per page, brand reference images are not shown in the generated proposal, the email does not expose analysis tags, `SECTION HTML 다운로드` creates a SECTION-only HTML file, and PDF output keeps three portfolio cards per page. For direct PDF export, verify `pdfinfo` reports A4 landscape page size and the expected page count.

## Matching Semantics

- Product candidates should prefer store/product-detail sources.
- Campaign/collaboration/project candidates should prefer Instagram or campaign-like sources.
- Similar recommendations use `대표무드형 매칭` by default: select a coherent set of 10 portfolio images that follows the brand's repeated cut types, tags, brightness, frame, and commerce roles.
- Whitespace recommendations focus on what the brand shows less often but the portfolio can offer as a new proposal direction.
- Tag chips in the HTML viewer are first-pass classifications. Treat them as review aids, not final truth.
